# 1.1

- SSI : Self-Sovereign Identity, 자기주권 신원증명

- 비밀키를 이용해 암호화를 수행하면 사용자 인증을 할 수 있음.
- 공개키를 이용해 암호화를 수행하면 원하는 상대에게만 데이터를 공개할 수 있음.


# 1.2

- DID, DID document : 식별자와 인증 수단
- VC (검증 가능한 자격증명) : 보관용 ID
- VP (검증 가능한 제공 ID 데이터 집합) : 제출용 ID
- 발행인(Issuer) : VC 발행
- 사용자 : VC 발급 후 VP로 가공하여 검증기관에 제출
- 검증인(Verifier) : 사용자로부터 VP를 수신하여 진위를 검증
- 블록체인 : DID 및 ID 관련 정보를 저장하는 분산 저장소


# 1.3

- SDK : Software Development Kit, 소프트웨어 개발 키트

- 하이퍼레저 인디 : 블록체인 노드와 클라이언트 간 연동을 위한 SDK 개발
 . indy-node : SSI 플랫폼에 특화된 블록체인 노드 개발
 . indy-plenum : indy-node 합의 알고리즘 개발
 . indy-sdk : indy-node와 연동하는 클라이언트 애플리케이션 개발용 SDK 개발

- 하이퍼레저 에어리즈 : SSI 관련 데이터를 처리하기 위한 P2P 프로토콜 개발
 . aries-rfcs : DID, VC, VP, 등 SSI 구성요소 관련 기술 사양 및 프로토콜 정의
 . aries-framework-go : Go 언어로 구현된 SSI 프레임워크 개발


# 2.1

- DID document : 인증 수단

- DID는 DID scheme, DID method, Method-specific identifier로 구성
 . DID scheme : URI가 어떤 프로토콜을 사용하는지 명시
 . DID method : DID document가 어떤 저장소에 저장되어 있는지 보여줌
 . Method-specific identifier : DID document가 저장된 정확한 위치 검색

- did:btcr:aaaa-bbbb-ccc
 . did : DID(URI) scheme
 . btcr : DID method, DID가 저장된 저장소 이름
 . aaaa-bbbb-ccc : Method-specific identifier, btcr 저장소 내 DID document가 저장된 주소
