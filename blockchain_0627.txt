- Vagrant란? 
 . 가상 시스템 환경을 구축하고 관리하기위한 도구
- VirtualBox에 Vagrant 설치 후 Indy-sdk 설치해보기 (~7/5 과제)


#. Vagrant 설치

1. powershell 관리자 권한으로 실행시킨 뒤 Chocolatey로 Vagrant 설치
 . choco install vagrant
 . 중간에 확인사항은 A 또는 Y 입력해서 동의

2. 설치 후 vagrant 명령어를 통해 명령어 확인


#. Vagrant 사용

1. PowerShell 경로를 Ubuntu 설치할 폴더로 이동

2. Vagrant를 사용할 초기 파일 생성
 - vagrant init

3. 생성된 Vagrantfile 15번째 줄 config.vm.box = "base" 내용을 아래 링크를 참고하여 수정
 - https://app.vagrantup.com/boxes/search
  . Ubuntu 18.04 desktop 버전으로 설치 (원본 : 16.04)
  . Provider에서 virtualbox 선택
  . 검색한 뒤 내용 수정
  . https://app.vagrantup.com/jpiel/boxes/ubuntu16.04-desktop 로 설치함

4. Powershell에 명령어 입력
 - vagrant up
  . 시간이 다소 걸릴 수 있음
  . 초기 user와 password는 vagrant

5. 실습 환경상 RAM은 4GB, CPU는 2개로 설정하고 진행 

6. 클립보드 공유를 통해 게스트 확장 설치
 . 장치 - 클립보드 공유 - 양방향
 . 장치 - 게스트 확장 CD 이미지 삽입...
  .. cd /media/vagrant/VBox_GAs_6.1.34/
  .. sudo ./VBoxLinuxAdditions.run 


#. Indy-sdk 설치

$ sudo apt-get install ca-certificates -y
$ sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys CE7709D068DB5E88
$ sudo add-apt-repository "deb https://repo.sovrin.org/sdk/deb xenial stable"
 - release channel은 master, rc, stable이 있음
  . master는 마스터 분기의 각 푸시에 대한 개발 빌드, rc는 후보 release, stable은 안정적인 release로 생각
  . stable로 설치 진행
$ sudo apt-get update
 - E: The method driver /usr/lib/apt/methods/https could not be found.
   N: Is the package apt-transport-https installed?
  . sudo apt-get install apt-transport-https ca-certificates -y
 - server certificate verification failed. CAfile: /etc/ssl/certs/ca-certificates.crt CRLfile: none
  . sudo apt-get install -y libgnutls30 (인증서 관련 라이브러리?)
$ sudo apt-get install libindy -y

- Ubuntu 16.04 버전에서는 설치가 잘 진행되지만 18.04에서는 libsodium18, 20.04에서는 (xenial|bionic) 부분에서 에러 발생
